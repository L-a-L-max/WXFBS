**文档版本**: vX.X.X

### 3. 图片引用规范

```markdown
<!-- 相对路径引用 -->
![架构图](./images/architecture/system-architecture.png)

<!-- 添加图片说明 -->
![登录页面截图](./images/screenshots/login-page.png)
*图1: 系统登录页面*
```

**图片规范**：
- 格式：PNG（截图）、SVG（图表）
- 命名：小写字母+连字符，如 `login-page.png`
- 存放：`docs/images/` 对应子目录
- 大小：单张图片不超过 500KB

### 4. 表格格式规范

**基本格式**：
```markdown
| 字段名  | 类型         | 说明     |
|--------|-------------|--------- |
| id     | BIGINT      | 主键ID   |
| title  | VARCHAR(500)| 文章标题  |
| status | TINYINT     | 状态     |
```

**显示效果**：

| 字段名 | 类型        | 说明     |
|--------|-------------|----------|
| id     | BIGINT      | 主键ID   |
| title  | VARCHAR(500)| 文章标题 |
| status | TINYINT     | 状态     |

**注意事项**：
- ✅ 使用竖线 `|` 分隔列
- ✅ 第二行用 `-` 和 `|` 定义表头分隔线
- ✅ 每列内容尽量对齐（可选，但更美观）
- ✅ 表格前后保持空行

## 📖 文档维护指南

### 1. 文档更新原则

- ✅ 及时更新：代码变更后及时更新相关文档
- ✅ 保持准确：确保文档内容与实际代码一致
- ✅ 简洁明了：避免冗长的描述，突出重点
- ✅ 图文并茂：适当使用图片和示例代码

### 2. 文档审查流程

1. **自查**：提交前检查格式、拼写、链接
2. **测试**：确保示例代码可运行
3. **提交**：遵循PR规范提交文档变更
4. **审查**：管理员审查文档质量
5. **合并**：审查通过后合并到主分支

### 3. 文档版本管理

```markdown
**最后更新**: 2025-12-05  
**文档版本**: v1.0.0
```

- 重大更新：主版本号+1（如：v1.0.0 → v2.0.0）
- 功能更新：次版本号+1（如：v1.0.0 → v1.1.0）
- 内容修正：修订号+1（如：v1.0.0 → v1.0.1）

### 4. 文档提交规范

遵循 [代码合并PR规范](./代码合并PR规范.md)：

```bash
# 提交消息示例
docs: 更新部署文档添加元器配置说明 (2025-12-05)
docs: 修正FAQ中的错误链接 (2025-12-05)
docs: 添加数据库设计规范文档 (2025-12-05)
```
